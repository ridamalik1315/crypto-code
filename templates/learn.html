How these ciphers work

XOR Cipher

XOR (exclusive-or) is a very simple symmetric operation: if two input bits are the same, the result is 0; if they are different, the result is 1. To encrypt, we XOR each bit of the plaintext with a key; to decrypt, we XOR the result with the same key again, which recovers the original text.

This is not secure on its own for real systems, but it is a great way to understand how stream ciphers and one-time pads work.

Permutation (Transposition) Cipher

A permutation cipher does not change the characters themselves; it only reorders them. We split the message into blocks and rearrange the positions inside each block according to a numeric key such as 2013.

To decrypt, we apply the inverse permutation, putting each character back in its original position. Modern block ciphers like AES also use permutations internally, combined with substitution steps.

RSA Public-Key Cipher

RSA is an asymmetric cipher: encryption and decryption use different keys. A public key (e, n) is used to encrypt, and a private key (d, n) is used to decrypt, where n = p Ã— q for large primes p and q.

Mathematically, encryption and decryption are just modular exponentiation: cipher = plain^e mod n and plain = cipher^d mod n. The difficulty of factoring n keeps the private key secure.

MD5 and Brute-Force Cracking

MD5 is a cryptographic hash function that maps any input to a 128-bit hash. It is fast but no longer considered secure due to collisions and its resistance to brute-force attacks being too low for modern hardware.

In this project, we brute-force all possible 3-character strings made of letters and digits, hash each one with MD5, and compare the result with a target hash. This illustrates how short, simple passwords can be discovered by attackers.
